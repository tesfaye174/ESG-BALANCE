Panoramica del Progetto

Cos'e' ESG-BALANCE?

Una piattaforma che unisce bilancio economico e indicatori ESG in un unico flusso: compilazione da parte del responsabile, revisione e approvazione finale.

Analisi della Ridondanza

nr_bilanci in aziende

Con ridondanza

  • OpA - Creazione bilancio: trigger aggiorna aziende (+1W)
  • OpB - Conteggio bilanci: lettura diretta da aziende (10R)
  • OpC - Rimozione azienda: trigger decrementa

Costo totale = 35

Senza ridondanza

  • OpA - Creazione bilancio: nessun costo extra
  • OpB - Conteggio: COUNT(*) su bilanci (50R)
  • OpC - Rimozione: nessun costo extra

Costo totale = 89

35 < 89 → Conviene mantenere la ridondanza. Gestita con Trigger T3 (INSERT +1) e Trigger T4 (DELETE -1).

06

Trigger

4 Trigger per automatizzare il ciclo di vita

T1: trg_bilancio_in_revisione
AFTER INSERT su revisioni
Quando si assegna il primo revisore, il bilancio passa da bozza a in_revisione
T2: trg_bilancio_giudizio
AFTER INSERT su giudizi
Quando tutti i revisori hanno votato:
  • Nessun respingimento → approvato
  • Almeno un respingimento → respinto
T3: trg_incrementa_nr_bilanci
AFTER INSERT su bilanci
Incrementa nr_bilanci dell'azienda di +1
T4: trg_decrementa_nr_bilanci
AFTER DELETE su bilanci
Decrementa nr_bilanci dell'azienda di -1

Vincoli e Regole di Business

Regole implementate

  • RV1 Livello competenza in [0,5] → CHECK
  • RV2 Rilevanza indicatore in [0,10] → CHECK
  • RV3 Ragione sociale univoca → UNIQUE
  • RV4 Primo revisore: bozza → in_revisione → Trigger T1
  • RV5 Stato finale da giudizi → Trigger T2
  • RV6 nr_bilanci aggiornato → Trigger T3/T4
  • RV9 Un giudizio per revisore per bilancio → PK composita
  • RV10 Modifica valori solo in stato bozza → logica PHP

Estratti dal Codice

Trigger T2 (stato finale bilancio)

CREATE TRIGGER trg_bilancio_giudizio AFTER INSERT ON giudizi FOR EACH ROW BEGIN IF NOT EXISTS ( SELECT 1 FROM revisioni r WHERE r.id_bilancio = NEW.id_bilancio AND NOT EXISTS ( SELECT 1 FROM giudizi g WHERE g.username_revisore = r.username_revisore AND g.id_bilancio = r.id_bilancio ) ) THEN UPDATE bilanci SET stato = CASE WHEN EXISTS ( SELECT 1 FROM giudizi WHERE id_bilancio = NEW.id_bilancio AND esito = 'respingimento' ) THEN 'respinto' ELSE 'approvato' END WHERE id = NEW.id_bilancio; END IF; END;

PHP: prepared statement + controllo stato

// Controllo che il bilancio sia in bozza $stmt = $pdo->prepare( "SELECT stato FROM bilanci WHERE id = ?" ); $stmt->execute([$idBilancio]); $stato = $stmt->fetchColumn(); if ($stato !== 'bozza') { throw new Exception('Solo in bozza'); } // Inserimento valore tramite SP $call = $pdo->prepare( "CALL sp_inserisci_valore_bilancio(?, ?, ?)" ); $call->execute([$idBilancio, $nomeVoce, $valore]);
