Panoramica del Progetto

Cos'è ESG-BALANCE?

Una piattaforma web che integra bilancio economico e indicatori ESG in un flusso unico: compilazione da parte del responsabile, revisione e approvazione finale.

02

I Tre Ruoli

Amministratore
  • Gestisce il template delle voci contabili
  • Definisce gli indicatori ESG (ambientali, sociali, generici)
  • Assegna i revisori ai bilanci
Responsabile Aziendale
  • Registra le aziende e carica il logo
  • Crea i bilanci e compila le voci contabili
  • Collega indicatori ESG alle voci
Revisore ESG
  • Dichiara le proprie competenze (livello 0-5)
  • Inserisce note sulle voci del bilancio
  • Emette il giudizio finale

05

Normalizzazione e Ridondanza

Tutte le 16 tabelle in BCNF — Ridondanza volontaria su nr_bilanci

Verifica BCNF (selezione)

Tabella Dipendenza Funzionale BCNF
utenti username → tutto Sì
aziende id → tutto; rag_soc → tutto Sì
bilanci id → tutto Sì
valori_bilancio (id_bil, voce) → valore Sì
voci_indicatori (id_bil, voce, ind) → val, fonte, data Sì
giudizi (user, id_bil) → esito, data, rilievi Sì

Nessuna decomposizione necessaria. Ogni determinante è superchiave.

  • OpB Conteggio: lettura diretta (10R)

Costo = 35

Senza ridondanza
  • OpB Conteggio: COUNT(*) su bilanci (50R)

Costo = 89

35 < 89 → ridondanza mantenuta con T3 (+1) e T4 (-1)

06

Trigger

4 Trigger per automatizzare il ciclo di vita

T1: trg_bilancio_in_revisione
AFTER INSERT su revisioni
Primo revisore assegnato → bilancio da bozza a in_revisione
T2: trg_bilancio_giudizio
AFTER INSERT su giudizi
Tutti i revisori hanno votato:
  • Nessun respingimento → approvato
  • Almeno un respingimento → respinto
  • Ricalcola indice_affidabilita (SP14)
T3: trg_incrementa_nr_bilanci
AFTER INSERT su bilanci
Incrementa nr_bilanci dell'azienda di +1
T4: trg_decrementa_nr_bilanci
AFTER DELETE su bilanci
Decrementa nr_bilanci dell'azienda di -1

Estratti dal Codice

Trigger T2 — stato finale bilancio

CREATE TRIGGER trg_bilancio_giudizio AFTER INSERT ON giudizi FOR EACH ROW BEGIN IF NOT EXISTS ( SELECT 1 FROM revisioni r WHERE r.id_bilancio = NEW.id_bilancioAND NOT EXISTS ( SELECT 1 FROM giudizi g WHERE g.username_revisore = r.username_revisore AND g.id_bilancio = r.id_bilancio ) ) THEN UPDATE bilanci SET stato = CASE WHEN EXISTS (...esito='respingimento') THEN 'respinto' ELSE 'approvato' END WHERE id = NEW.id_bilancio; END IF; -- Aggiorna indice affidabilita CALL sp_aggiorna_indice_affidabilita( NEW.username_revisore); END;

PHP — prepared statement + controllo stato

// Controllo che il bilancio sia in bozza $stmt = $pdo->prepare( "SELECT stato FROM bilanci WHERE id = ?" ); $stmt->execute([$idBilancio]); $stato = $stmt->fetchColumn(); if ($stato !== 'bozza') { throw new Exception('Solo in bozza'); } // Inserimento valore tramite SP $call = $pdo->prepare( "CALL sp_inserisci_valore_bilancio(?, ?, ?)" ); $call->execute( [$idBilancio, $nomeVoce, $valore] );
