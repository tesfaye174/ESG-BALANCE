Panoramica del Progetto

ESG-BALANCE è un'applicazione web che permette di gestire i bilanci aziendali e collegarli agli indicatori ESG (Environmental, Social, Governance). Prevede tre ruoli utente e un processo di revisione con approvazione o respingimento.

🛡 Amministratore
  • Gestisce template voci contabili
  • Crea e categorizza indicatori ESG
  • Assegna revisori ai bilanci
  • Monitora statistiche globali
🔍 Revisore ESG
  • Esamina i bilanci assegnati
  • Inserisce note di revisione
  • Emette giudizio finale
  • Indice di affidabilità automatico
🏢 Responsabile
  • Gestisce le proprie aziende
  • Crea e compila bilanci annuali
  • Collega indicatori ESG alle voci
  • Monitora lo stato revisioni

Analisi della Ridondanza — nr_bilanci

Tavola dei Volumi

ConcettoTipoVolume
aziendeEntità10
bilanciEntità50 (5 per azienda)

Tavola delle Operazioni

#OperazioneFreq.Tipo
OP1Aggiungere azienda + 3 bilanci1/meseI
OP2Contare bilanci per tutte le aziende3/meseB
OP3Rimuovere azienda + bilanci connessi1/meseB
ℹ

Parametri: wI = 1, wB = 0.5, a = 2 (costo scrittura = 2 × lettura)

Costo Totale Mensile

Op. Con ridondanza Senza ridondanza
OP1 1Waz + 3Wbil + 3Waz(T3)
(2 + 6 + 6) × wI × 1 = 14 		1Waz + 3Wbil
(2 + 6) × wI × 1 = 8
OP2 10Raz (legge nr_bilanci)
10 × wB × 3 = 15 		10Raz + 50Rbil (COUNT JOIN)
60 × wB × 3 = 90
OP3 1R+1Waz + 5Wbil + 5Waz(T4)
(3 + 10 + 10) × wB × 1 = 11.5 		1R+1Waz + 5Wbil (cascade)
(3 + 10) × wB × 1 = 6.5
TOTALE 40.5 104.5

Decisione: si mantiene la ridondanza (40.5 < 104.5).
Il risparmio su OP2 (lettura batch 3×/mese) è determinante: la JOIN completa su 50 bilanci costa 6× di più rispetto alla semplice lettura del campo nr_bilanci. Il trigger T3/T4 gestisce l'aggiornamento automatico senza intervento applicativo.

Gerarchia Utenti

SQL
CREATE TABLE utenti ( username VARCHAR(50) PRIMARY KEY, password_hash VARCHAR(255) NOT NULL, codice_fiscale CHAR(16) NOT NULL, data_nascita DATE NOT NULL, luogo_nascita VARCHAR(100) NOT NULL, ruolo ENUM('amministratore','revisore','responsabile'), UNIQUE KEY (codice_fiscale) ); CREATE TABLE revisori ( username VARCHAR(50) PRIMARY KEY, nr_revisioni INT DEFAULT 0, indice_affidabilita DECIMAL(3,2) DEFAULT 0.00, FOREIGN KEY (username) REFERENCES utenti(username) ON DELETE CASCADE );

Aziende e Bilanci

SQL
CREATE TABLE aziende ( id INT AUTO_INCREMENT PRIMARY KEY, ragione_sociale VARCHAR(200) UNIQUE NOT NULL, partita_iva VARCHAR(11) UNIQUE NOT NULL, settore VARCHAR(100), nr_bilanci INT DEFAULT 0, username_responsabile VARCHAR(50), FOREIGN KEY (username_responsabile) REFERENCES responsabili(username) ); CREATE TABLE bilanci ( id INT AUTO_INCREMENT PRIMARY KEY, id_azienda INT NOT NULL, anno YEAR NOT NULL, data_creazione DATE NOT NULL, stato ENUM('bozza','in_revisione','approvato','respinto'), UNIQUE KEY (id_azienda, anno), FOREIGN KEY (id_azienda) REFERENCES aziende(id) );

Trigger — 4 totali

T1 — Cambio stato: bozza → in revisione

SQL
CREATE TRIGGER trg_bilancio_in_revisione AFTER INSERT ON revisioni FOR EACH ROW BEGIN UPDATE bilanci SET stato = 'in_revisione' WHERE id = NEW.id_bilancio AND stato = 'bozza'; END;

T3/T4 — Ridondanza nr_bilanci

SQL
-- T3: AFTER INSERT ON bilanci CREATE TRIGGER trg_incrementa_nr_bilanci AFTER INSERT ON bilanci FOR EACH ROW BEGIN UPDATE aziende SET nr_bilanci = nr_bilanci + 1 WHERE id = NEW.id_azienda; END; -- T4: AFTER DELETE ON bilanci CREATE TRIGGER trg_decrementa_nr_bilanci AFTER DELETE ON bilanci FOR EACH ROW BEGIN UPDATE aziende SET nr_bilanci = GREATEST(nr_bilanci - 1, 0) WHERE id = OLD.id_azienda; END;

T2 — Chiusura dopo tutti i giudizi

SQL
CREATE TRIGGER trg_bilancio_giudizio AFTER INSERT ON giudizi FOR EACH ROW BEGIN DECLARE v_tot_revisori INT; DECLARE v_tot_giudizi INT; DECLARE v_respingimenti INT; SELECT COUNT(*) INTO v_tot_revisori FROM revisioni WHERE id_bilancio = NEW.id_bilancio; SELECT COUNT(*) INTO v_tot_giudizi FROM giudizi WHERE id_bilancio = NEW.id_bilancio; IF v_tot_giudizi = v_tot_revisori THEN SELECT COUNT(*) INTO v_respingimenti FROM giudizi WHERE id_bilancio = NEW.id_bilancio AND esito = 'respingimento'; IF v_respingimenti > 0 THEN UPDATE bilanci SET stato='respinto' WHERE id = NEW.id_bilancio; ELSE UPDATE bilanci SET stato='approvato' WHERE id = NEW.id_bilancio; END IF; END IF; CALL sp_aggiorna_indice_affidabilita( NEW.username_revisore ); END;

SP14 — Aggiornamento indice affidabilità

SQL
CREATE PROCEDURE sp_aggiorna_indice_affidabilita( IN p_username VARCHAR(50) ) BEGIN DECLARE v_totale INT DEFAULT 0; DECLARE v_positivi INT DEFAULT 0; SELECT COUNT(*) INTO v_totale FROM giudizi WHERE username_revisore = p_username; SELECT COUNT(*) INTO v_positivi FROM giudizi WHERE username_revisore = p_username AND esito = 'approvazione'; IF v_totale > 0 THEN UPDATE revisori SET indice_affidabilita = ROUND(v_positivi / v_totale, 2) WHERE username = p_username; END IF; END;

Query di Interrogazione

Q1 — Bilanci in revisione

SQL
SELECT b.id, a.ragione_sociale, b.anno, b.stato FROM bilanci b JOIN aziende a ON b.id_azienda = a.id WHERE b.stato = 'in_revisione' ORDER BY b.data_creazione DESC;

Q2 — Revisori con competenze (livello ≥ 3)

SQL
SELECT cr.username, cr.nome_competenza, cr.livello FROM competenze_revisore cr WHERE cr.livello >= 3 ORDER BY cr.livello DESC;

Q3 — Indicatori ESG ambientali collegati

SQL
SELECT ie.nome AS indicatore, ie.tipo, vi.valore_indicatore, vi.nome_voce AS voce FROM voci_indicatori vi JOIN indicatori_esg ie ON vi.nome_indicatore = ie.nome WHERE ie.tipo = 'ambientale';

Q4 — Note di revisione per bilancio

SQL
SELECT u.codice_fiscale, nr.testo, nr.data_nota FROM note_revisione nr JOIN utenti u ON nr.username_revisore = u.username WHERE nr.id_bilancio = ?;

SQL
CREATE VIEW v_num_aziende AS SELECT COUNT(*) AS totale_aziende FROM aziende; CREATE VIEW v_num_revisori AS SELECT COUNT(*) AS totale_revisori FROM revisori;

V4 — Classifica bilanci per indicatori ESG

SQL
CREATE VIEW v_classifica_bilanci_esg AS SELECT b.id AS id_bilancio, a.nome AS azienda, b.anno, b.stato, COUNT(vi.nome_indicatore) AS num_indicatori_esg FROM bilanci b JOIN aziende a ON a.id = b.id_azienda LEFT JOIN voci_indicatori vi ON vi.id_bilancio = b.id GROUP BY b.id, a.nome, b.anno, b.stato ORDER BY num_indicatori_esg DESC;

V3 — Classifica affidabilità aziende

SQL
CREATE VIEW v_affidabilita_aziende AS SELECT a.id AS id_azienda, a.nome, a.ragione_sociale, COUNT(DISTINCT b.id) AS bilanci_giudicati, COUNT(DISTINCT CASE WHEN NOT EXISTS ( SELECT 1 FROM giudizi g2 WHERE g2.id_bilancio = b.id AND g2.esito <> 'approvazione' ) THEN b.id ELSE NULL END) AS bilanci_approvati_puri, ROUND( COUNT(DISTINCT CASE WHEN ... END) * 100.0 / COUNT(DISTINCT b.id), 2 ) AS percentuale_affidabilita FROM aziende a JOIN bilanci b ON b.id_azienda = a.id JOIN giudizi g ON g.id_bilancio = b.id GROUP BY a.id, a.nome, a.ragione_sociale ORDER BY percentuale_affidabilita DESC;
